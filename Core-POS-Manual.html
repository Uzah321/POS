ℹ️
Core POS runs entirely offline. No internet connection is needed after the initial installation. All data is stored locally in a MariaDB database on the same machine.

1.2 Installation Steps

1
Copy the installer to the target machine Copy Core-Setup.exe to the machine. A USB drive works fine — no internet is needed.
2
Run as Administrator Right-click Core-Setup.exe → Run as administrator. If prompted by Windows Defender or UAC, click Yes / Run anyway.
3
Follow the Setup Wizard Accept the default installation path (C:\POS) and click Next through the wizard pages. No options need to be changed.
4
Wait for automatic configuration (2–5 minutes) A setup window opens and automatically:
  • Installs the Visual C++ Runtime (required by PHP)
  • Installs MariaDB database silently
  • Creates the corepos database and application user
  • Runs all database migrations and seeds default data
  • Creates a Core shortcut on the Desktop
✅
The window will say "Core POS setup complete!" when finished. Press Enter to close it.
5
Launch Core POS Double-click the Core shortcut on the Desktop, or tick "Launch Core now" at the end of the wizard. Your browser opens automatically at http://localhost:8080.

Finding the Server IP Address

On the POS machine, open PowerShell or Command Prompt and type:

ipconfig
# Look for "IPv4 Address" under your active network adapter
# Example output:
   IPv4 Address. . . : 192.168.1.10

Then on any device on the same network, open a browser and go to http://192.168.1.10:8080/kitchen (replace with your actual IP).

1.5 Firewall Configuration

Windows Firewall may block connections from other devices. Run this command once on the POS server machine (as Administrator):

netsh advfirewall firewall add rule name="Core POS" dir=in action=allow protocol=TCP localport=8080

This only needs to be done once. After this, all devices on the local network can connect to the POS server.

1.6 Starting the System Daily

Each day, simply double-click the Core shortcut on the Desktop. It will:

  • Start the MariaDB database service automatically
  • Launch the PHP web server on port 8080
  • Open your browser at the cashier screen
💡
Leave the black command window open while the system is in use. Closing it will stop the POS server. All other screens (kitchen, queue, cashier terminals) will stop working until you re-launch.

2.3 Advanced POS — Restaurant

The Advanced POS (/pos) is designed for table-service restaurants. It supports table management, modifiers, order routing to the kitchen, and split payments.

Starting a Sale

1
Select or create a table / order — Click an available table on the floor plan, or press New Order for takeaway.
2
Add items — Search by name or scan a barcode. Products are grouped by category. Click an item to add it to the order.
3
Apply modifiers / notes — Tap a line item to adjust quantity, add a note (e.g. "no chilli"), or apply a discount.
4
Send to Kitchen — Click Send to KDS. The order appears instantly on the Kitchen Display Screen.
5
Collect payment — When the customer is ready, click Charge. Select the payment method and confirm. A receipt is generated.

Payment Methods Accepted

💵
Cash
Enter amount tendered. System calculates change automatically.
💳
Card
Credit/debit card via external terminal. Mark as card payment in the system.
📱
EcoCash
Mobile money payment. Enter reference number and confirm.
🔀
Split Payment
Customer pays part cash, part card — handled within a single transaction.

Holding / Parking a Sale

To pause a sale and start a new one (e.g. while waiting for a customer to return), click Hold Order. The order is saved and can be recalled from the Held Orders list.

Applying Discounts

  • Line discount — Tap a product line and enter a % or fixed discount on that item only.
  • Order discount — Apply a % discount to the entire order at the payment screen.
  • Customer discount — Customers with a discount rate on their profile get it automatically when linked to the sale.

2.4 Cashier Register — Supermarket

The Cashier Register (/cashier) is optimised for high-volume, fast-paced supermarket checkout. It supports barcode scanning, quick product search, and rapid payment processing.

Cashier Workflow

1
Scan or search products — Use a barcode scanner or type in the search box. Products are added to the basket instantly.
2
Review basket — The right panel shows all items, quantities, and the running total including tax.
3
Link a customer (optional) — Search by name or phone number to apply loyalty points or a customer discount.
4
Charge — Click Charge, select payment method(s), and confirm. Receipt prints automatically if a printer is connected.

Price Check

Scan any item barcode without adding it to the basket by pressing the Price Check button first. The price is displayed without affecting the current sale.

Returns & Refunds

To process a return, go to Sales, find the original sale, click View, and select Refund. Choose which items to return and the refund method. Stock is automatically added back to inventory.

Layby Sales

A customer can reserve goods by paying a deposit. Go to Laybys or click Layby on the cashier screen. Enter the deposit amount. The customer returns later and pays the balance to collect the goods.

2.5 Kitchen Display System & Queue Screen

ℹ️
Both screens are public — they require no login and open directly in a browser. They are designed to run on a dedicated screen or TV, always visible to kitchen staff or customers.

Kitchen Display Screen (KDS) — /kitchen

The KDS shows all active orders sent from the POS. Each order card displays the ticket number, table/reference, and list of items. Kitchen staff bump orders through the workflow:

StatusMeaningCard Colour
NewOrder just received, not yet startedBlue border
PreparingKitchen is actively cooking this orderAmber border
ReadyOrder is ready for collection / serviceGreen border
ServedOrder delivered to customer — removed from screenHidden

Bumping an Order

Click the button at the bottom of each order card to advance it to the next status. The button label changes automatically: Bump to Preparing → Mark Ready → Served.

Auto-refresh

The KDS polls for new orders every 6 seconds. No manual refresh is needed. An audio notification plays when a new order arrives (browser must allow sound).

Queue Display Screen — /queue

The Queue Display shows order numbers on a TV visible to waiting customers. When an order is marked Ready on the KDS, the ticket number appears on the queue board. When the customer collects and the order is marked Served, the number is removed.

Accessing KDS from another device

  1. Find the POS server's IP address (see Section 1.3)
  2. Open a browser on the kitchen screen device
  3. Navigate to http://[SERVER-IP]:8080/kitchen
  4. Bookmark the page or set it as the browser's home page

Products (/products)

The Products page is the master catalogue of everything sold in your business.

FieldDescription
NameProduct name displayed on receipts and the POS screen
Barcode / SKUUsed for barcode scanning. Can be auto-generated.
CategoryGroups products in the POS product grid
Selling PricePrice charged to the customer (tax-inclusive or exclusive per settings)

Inventory (/inventory)

Shows current stock levels for all products with inventory tracking enabled. Columns include: product name, current quantity, minimum level, and unit of measure.

Adding Stock

Stock is added automatically when you record a Purchase (see Section 2.7). You can also manually adjust stock by clicking the Adjust button on any inventory row — useful for corrections after a physical count.

Stocktake (/stocktake)

Use this when doing a full physical count. Enter the counted quantity for each product. The system calculates variances (differences between expected and actual stock) and lets you submit the count to update inventory.

Stock Transfer (/stock-transfers)

Transfer stock between branches. Select the source branch, destination branch, product, and quantity. The transfer is logged and both branches' inventory is updated.

Stock Reconciliation (/stock-reconciliation)

Reconcile discrepancies between system stock and physical count. Records the reason for adjustments (wastage, theft, damage) for audit purposes.

2.9 Sales & Orders

Sales History (/sales)

A complete log of every completed sale. You can filter by date range, cashier, branch, or payment method. Each sale can be:

  • Viewed — see full itemised breakdown and payment details
  • Re-printed — reprint the receipt to the configured printer
  • Refunded — process a full or partial return
  • Exported — download as CSV or PDF

Orders (/orders) — Restaurant only

The Orders page shows all table orders in a restaurant context, including open orders, held orders, and recently completed orders. Staff can reassign orders to tables or merge tickets from here.

My Sales (/my-sales)

Each cashier can view only their own sales for the current shift. This page is accessible to all staff roles and is used for self-reconciliation at shift end.

Quotations (/quotations)

Generate a price quotation for a customer without completing a sale. Quotations can be saved, printed, and later converted into a completed sale with one click.

Laybys (/laybys)

A layby (layaway) allows a customer to reserve goods by paying a deposit. Key actions:

  • New layby — create from the cashier screen or from the Laybys page
  • Add payment — record partial payments against the layby balance
  • Complete — when fully paid, convert to a completed sale
  • Cancel — refund and release stock back to inventory

Cashflow (/cashflow)

Tracks all money moving in and out of the business. Shows a timeline of sales income versus expenses, giving a net cash position per day and per month. Use this to monitor whether the business is cash-positive.

  • Cost of Goods Sold (COGS)

Salaries (/salaries)

Manage staff payroll. Set up each employee's salary, record monthly payroll runs, and track payment status. Salary payments are linked to the expense system so they appear in the P&L automatically.

Commissions (/commissions)

Set commission rates per product or category for specific sales staff. The system calculates earned commissions based on sales. Commission reports can be run per employee per period.

EcoCash (/ecocash)

Dedicated EcoCash (mobile money) management. View all EcoCash transactions, reconcile payments, and export for reporting to mobile money providers.

Rentals (/rentals)

For businesses that rent out equipment or space. Create rental agreements with start/end dates and rates. The system tracks active rentals and generates invoices automatically on the due date.

2.12 Shift End & Day End

Shift End (/shift-end)

Each cashier runs a Shift End when their shift finishes. The process:

1
System summary — Shows total sales, expected cash in drawer, and total by payment method for the shift.
2
Count cash — Enter the actual notes and coins counted in the drawer.
3
Variance — System shows the difference (over/short). Provide a reason if there is a discrepancy.
4
Submit — Shift is closed. Manager is notified if there is a significant variance.

Day End (/day-end)

The Day End is run by a manager at the close of business. It consolidates all shifts for the day, produces the end-of-day report, and resets the daily counters. The report includes total sales, total cash, all payment methods, and a reconciliation summary.

2.13 Branches

Core POS supports multiple business locations (branches) from a single installation. Branches share the same product catalogue and user base but have separate sales, inventory, and financial data.

Managing Branches (/branches)

Create and manage branches from the Branches page (admin only). Each branch has a name, address, and contact details. The active branch is shown in the top navigation bar.

Branch Filtering

On all reports, sales, and inventory pages, a Branch filter dropdown allows managers to view data for one branch or across all branches simultaneously.

ℹ️
Stock transfers between branches are handled on the Stock Transfers page. Inventory levels are maintained independently per branch.

Set up tax rates (e.g. VAT 15%) from Settings. Tax can be applied per product or as a store-wide default. Tax is calculated at the point of sale and reported separately in the Tax Report.

Switching Business Type

To switch from Restaurant to Supermarket (or vice versa), go to Settings → General, change the Business Type field, and save. The sidebar navigation and dashboard will update immediately on next page load.

Webhooks (/webhooks)

Webhooks allow Core POS to notify external systems (e.g. accounting software, loyalty apps) when events occur, such as a new sale or a refund. Configure the target URL and select which events to send. The system sends a JSON payload to the URL on each event.

2.16 Hardware & Peripherals

Configure connected hardware from /hardware.

Receipt Printer

Core POS supports ESC/POS thermal receipt printers connected via USB or network. Configure the printer type and connection in Hardware Settings. Test print from the same page. Receipts print automatically on each completed sale.

Barcode Scanner

USB barcode scanners work as keyboard input devices — plug in and scan. No driver or configuration needed. The scanner input is captured by the product search field on both the POS and Cashier screens.

Cash Drawer

If connected via the receipt printer's RJ11 port, the cash drawer opens automatically when a cash payment is completed. Configure the drawer kick pulse in Hardware Settings.

Customer Display

A secondary screen connected to the cashier PC can show the running sale to the customer at /customer-display. It updates in real-time as items are scanned.

2.17 Backup & Restore

🔴
Back up your data regularly. A hardware failure without a backup means all sales history, customer records, and product data is permanently lost. We recommend a daily backup to an external USB drive or cloud storage.

Creating a Backup (/backups)

1
Go to Backups from the sidebar (admin only).
2
Click "Create Backup". The system exports all data to a compressed file.
3
Download the backup file and copy it to a USB drive or email it to yourself. Store it off the POS machine.

Restoring from Backup

1
Go to Backups and click Restore.
2
Upload the backup file.
3
Confirm the restore. The system will restart automatically. All data is replaced with the backup.
⚠️
Restore overwrites all current data. Any sales made after the backup date will be lost. Only restore from backup when recovering from a failure or migrating to a new machine.

2.18 Advanced Features

Audit Logs (/audit-logs)

Every significant action in Core POS is recorded in the audit log: logins, sales, refunds, product changes, user changes, and settings modifications. Each entry shows the user, action, timestamp, and what changed. This provides a full accountability trail for compliance and dispute resolution.

Stocktake

A formal inventory count. The system generates a list of all stocked products. Staff enter the physical count per product. After submission, the system updates inventory quantities to match the count and records any variances.

Recommended practice: Run a stocktake at the end of each month and after any major delivery.

Stock Reconciliation

If a variance is found during stocktake or daily operations, use Stock Reconciliation to formally record the adjustment with a reason (theft, wastage, damage, data error). This maintains the integrity of your inventory audit trail.

Currencies — Multi-Currency Sales

When a customer pays in a foreign currency, select the currency at the payment screen. The system converts the amount at the configured exchange rate and records the sale in the base currency for reporting purposes.

Webhooks & Integrations

Core POS can push real-time event notifications to external systems. Supported events include: sale.completed, sale.refunded, product.updated, customer.created. Each webhook fires a POST request with a JSON payload to your configured endpoint URL.
