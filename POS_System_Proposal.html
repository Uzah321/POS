11 · Offline Mode & Business Continuity

Never lose a sale — even when your internet connection does

"In markets where internet reliability is inconsistent, the ability to keep trading during outages is not a nice-to-have — it is a business requirement. NexaPOS was built with this reality in mind."
📡

Automatic Offline Detection

NexaPOS detects loss of internet connectivity instantly and switches to offline mode — no action required from staff.

🛒

Offline Sales Processing

Process complete sales while offline. Products are served from the local browser cache — no blank screens or errors.

📥

Offline Receipt Printing

Receipts print normally during outages, generated from local cart data so customers always get proof of purchase.

☁️

Automatic Sync on Reconnect

When internet is restored, all queued offline sales upload to the server automatically — no manual intervention.

🔁

Manual Sync Button

A "Sync Now" button allows staff to trigger upload immediately after reconnection if preferred.

🔢

Pending Sale Counter

An amber banner shows the number of queued offline sales so management always knows what's pending.

How it works technically: NexaPOS is a Progressive Web App (PWA). Product catalogues, settings, currencies, and warehouse data are cached by a Workbox service worker. Offline sales are stored in the browser's local storage and posted to the server with an is_offline flag once connectivity is restored. The system is built on React 19 + TanStack Query with offlineFirst network mode, meaning cached data always serves from local cache when the network is unavailable.

Full accountability — every action recorded, every screen controlled

Role-Based Access Control

  • Unlimited user accounts per branch
  • Pre-built roles: Admin, Manager, Cashier, Stock Clerk, and more
  • Granular permission toggle per module per action (view, create, edit, delete)
  • Permission matrix visible and editable in the admin dashboard
  • Branch-scoped access — staff see only what they need
  • Token-based authentication via Laravel Sanctum

Audit Log

  • Immutable log of every create, edit, and delete in the system
  • Every entry records: user, action, affected entity, before/after values, and timestamp
  • Searchable and filterable by user, date, or action type
  • Cannot be modified or deleted — even by administrators
  • Webhook alerts for critical events (sales, refunds, low stock)
  • Automated database backup for disaster recovery

13 · Technology Stack & Architecture

Built on proven, enterprise-grade open-source foundations

Back-End API

PHP 8.3+ Laravel 13 Laravel Sanctum (Auth) Eloquent ORM PostgreSQL RESTful JSON API Spatie Permissions (RBAC) Queued Jobs Webhook Dispatching

Front-End PWA

React 19 TypeScript Vite 6 Tailwind CSS TanStack Query v5 Zustand (State) Recharts (Dashboards) Workbox (Service Worker) Lucide Icons

NexaPOS follows a decoupled API architecture — the React front-end communicates with the Laravel API over HTTPS with bearer token authentication. The front-end is a Progressive Web App (PWA), meaning it can be installed to the home screen of any device and works offline. The system is designed to run on any Linux VPS, dedicated server, or local network — no proprietary cloud lock-in. All data is stored in a PostgreSQL relational database with full referential integrity and foreign key constraints.

16 · Implementation Roadmap

From signed proposal to fully live system in as little as one week

Discovery & Requirements (Day 1)

We meet with your team to document your product catalogue, user roles, branch structure, hardware inventory, currency setup, and any specific workflows. A configuration checklist is agreed and signed off.

Server Installation & Configuration (Day 2)

NexaPOS is deployed to your server or VPS, connected to your domain with SSL, and configured with your branding, logo, currency, tax settings, branch structure, and user roles.

Data Migration (Day 3–4)

Your existing product catalogue, customer records, supplier list, and opening stock levels are imported from your current system or spreadsheets. Accuracy is verified before proceeding.

Hardware Setup & Testing (Day 5)

Receipt printers, barcode scanners, cash drawers, and customer displays are connected and tested end-to-end with actual counter hardware. A full test sale is processed to confirm everything works.

Staff Training (Day 6)

Practical, hands-on training for cashiers, managers, and administrators. We cover the daily operating workflow from opening to end-of-day close, plus a printed quick-reference guide for every station.

Go Live & Hypercare (Day 7+)

The system goes live. Our team is on standby for the first week (WhatsApp / call) to resolve any questions and ensure a smooth transition. Support continues for the duration of your support package.
